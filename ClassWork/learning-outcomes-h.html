Serverside and Clientside Programming

Server-side programming:

The program code runs from the server hosting the website

Advantage

Connect to a server from an online database containing info not directly accessible to end users

Disadvantages

Use server resources and require internet access
Long delays in cases of load

Client-Side Programming

The program code runs on the user's computer using downloaded scripts with html and CSS files.
The load gets distributed avoiding program-related requests Client-side programs can never replace server-side programming.

Starting with JavaScript

Code is attached to an html file using the [script] element
[script src ="url"][/script]

An embedded script can be used instead of an external file
[script]
code
[/script]

Script elements can be placed anywhere in an HTML document.
When encountered the browser halts loading and starts proccessing the script.
[async] and [defer] are attributes that can be added to [script] to alter the sequence of proccessing.

async: Parse the HTML and JavaScript code together.
defer: defers script processing until after the page has been completely parsed and loaded.
NOTE: These attributes are ignored for embedded scripts.

Debugging

Common Errors

Load-time errors occur when a script is first loaded by a browser
Run-time errors occur during execution of a script without syntax errors
Logical errors are free from syntax and executable mistakes but result in an incorrect output

Debugging Tools

F12 is the browser shortcut to open a debugger

Inserting breakpoints is a usefull technique to locate the source of an error.
Breakpoints are locations where a browser pauses a program to determine whether an error has occurred at that point during execution .
Insert a breakpoint using a simple window.alert("") function.

Applying Strict Usage of JavaScript

strict mode enables all lapses in syntax to result in load-time or run-time errors.
To run in strict mode , add the ["use strict";] statement to the first line of the file;

Objects

Object: Entity within a browser or webpage that has properties and methods.
Properties: Define objects.
Methods: Act upon objects .
Java is an object-based language that manipulates and object by changing one or more of its properties.

Types of JS Objects

Built-in intrisic to JS language
Browser part of browser
Document part of web documents
Customised creatred by a programmer to use in an application

Browser object model (BOM) and Document object model(DOM) organize browser and document objects in hierachical structures , respectively.

Object hierachy

Objects within the Obj hierarchy are referenced by their obj names such as window , document or navigator.
Objects can be referenced using the notation [object1. object2.object3...] where object1 is at the top of the hierarchy , object2 is a child of object1 , and so on.

Referencing an Object by ID and Name

An efficient approach to reference an element is to use its id attribute using the expression:
[document.getElementById(id)]
where id is the value of the id ,id="example-container"

Changing Properties and Applying Methods

Object Properties

Object properties is accessed using [object.property]
Read-only properties cannot be modified.

Applying a Method

Objects can be modified using methods.
Methods are applied using the expression : [object.method(values)] where object is a reference to an object , method is the name of the method applied to the object , and value is a comma-seperated list of values is associated with the method.

Writing HTML code

HTML code stored within a page element is referenced using:
[element.innerHTML] Where element is an obj reference to and element within a web document.

Example:

document.getElementById("dateNow").innerHTML="m/d/y h:m:s"
